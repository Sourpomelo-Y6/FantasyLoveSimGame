# Codex / ChatGPT を使った Unity 2D 開発手法

このプロジェクトでは、Codex CUI と ChatGPT を併用して Unity 2D ゲームを開発しています。
コードや ScriptableObject などのデータ作成は Codex に任せつつ、Unity Editor の GUI 上で行う UI 配置は人間が手作業で行う方針です。

## 基本方針

- Codex CUI だけでもコードやアセットファイルの作成は進められる
- ただし、Unity Editor の GUI 上に自動で UI 部品が配置されるわけではない
- そのため、UI の Hierarchy 配置や Inspector の参照割り当ては人間が手作業で行う
- Codex / ChatGPT には、手作業で何を設置し、どこに何を割り当てるかを具体的に指示してもらう
- ソースコード、データ定義、ドキュメント、コミットは Codex に任せる

少なくともこのプロジェクトで使っている Unity `2021.3.45f1` では、Codex が作った UI コードや Scene YAML の変更だけでは、Unity Editor 上の手触りが分かりにくいことがあります。
そのため、UI は「コードで生成する」のではなく「Unity 上に手で配置して、コードから参照する」形を基本にしています。

## プロジェクト開始時の進め方

Web 版 ChatGPT に、次のように依頼します。

```text
Unity で簡単な○○ゲームを作りたいです。
GUI 部分はコードではなく手作業で作ります。
まずは最小限動く形を提案してください。
```

最初は、できるだけ小さく動くものを作ります。
簡単に動くものができたら、`Docs` フォルダを作ってもらい、追加したい機能や将来の構想をドキュメントに残していきます。

例:

```text
この機能も後で入れたいので、Docs 内の開発計画に残してください。
```

会話が長くなると抜けが発生しやすくなるため、区切りごとに引き継ぎ用ドキュメントも書いてもらいます。
その後、WSL2 上の Codex CUI に移ってもよいです。

## Codex CUI で続きを進める場合

以前は、作業開始時に毎回次のように依頼していました。

```text
Docs フォルダ内の内容を確認して、このプロジェクトの概要を教えてください。
```

現在は会話 ID などで引き継ぎやすくなっていますが、長く間が空いた場合や別モデルに切り替える場合は、Docs を確認させると安定します。

日常的な進め方は次の流れです。

```text
次にすることは何ですか？
```

Codex / ChatGPT が次の作業を 1 つ、または複数提示します。
その後、次のように依頼します。

```text
そのまま進めてください。
```

または、

```text
2 を進めてください。
```

実装後は Unity Editor 上で人間が手動確認します。
確認できたら、次のように依頼します。

```text
動作確認しました。コミットをお願いします。
```

## 思いついた機能や不具合の扱い

作業中に思いついた機能は、すぐ実装する場合と、ドキュメントだけに残す場合を分けます。

```text
このアイディアを Docs 内に残してください。
```

```text
この仕様で実装してください。
```

バグや仕様違いも同様に、まず Codex に状況を伝えます。
うまく直らない場合は、次のどれかに切り替えます。

- `gpt-5.5` または `gpt-5.6` に切り替える
- 作業を小さく分解して依頼する
- Unity Editor 上の手作業で直す
- いったんドキュメントに残して後回しにする

## 区切りのよいタイミング

終了時や、機能がひと段落したタイミングでは、ドキュメント更新を依頼します。

```text
ドキュメントの更新をお願いします。
```

実装内容、未解決の課題、次にやるべきことを残しておくと、次回の再開が安定します。

## 画像アセットの扱い

キャラクター画像、背景、イベントスチルなどは ChatGPT Images 2.0 や Stable Diffusion で用意します。
画像を用意した後は、Codex / ChatGPT に次のように聞きます。

```text
絵を用意したので、Unity で表示するにはどうしますか？
```

その後、画像の置き場所、ファイル名、Texture 設定、UI の Image への割り当て、コード側の参照方法を整理してもらいます。

このプロジェクトでは、画像そのものの生成と、Unity 上での UI 配置は人間が確認しながら進め、コードやデータへの組み込みを Codex に任せる方針です。
