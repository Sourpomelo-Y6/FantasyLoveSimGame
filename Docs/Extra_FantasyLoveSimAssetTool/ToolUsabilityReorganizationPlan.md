# AssetTool 操作整理案

AssetToolの操作画面は、機能別タブではなく利用者が完了させたい作業を基準に整理する。
詳細な提案の正本は、AssetToolリポジトリの `Docs/ToolUsabilityReorganizationPlan.md` とする。

## 推奨構成

主タブを次の4つへ整理する。

1. `キャラクター`: ヒロイン、プレイヤー、敵の選択と基本情報
2. `画像制作`: 立ち絵、戦闘、訓練、イベント、行動、エンディングの登録と生成
3. `ゲームデータ`: 会話、イベント、行動反応、表情・衣装・レイヤー定義
4. `確認・Export`: 不足、保留、警告、Export対象と結果

ComfyUI接続設定やworkflow template編集は、日常操作から分離して `設定・詳細` に置く。

## 画像制作の共通手順

```text
対象選択 → 必要画像選択 → Prompt確認 → ComfyUI生成または外部画像指定 → Preview → 採用
```

- 左側に必要画像と登録済み画像をまとめて表示する。
- 中央にPromptと生成操作を表示する。
- 右側に現在画像、生成候補、検査結果、採用操作を表示する。
- 外部登録とComfyUI生成は同じAsset、Prompt JSON、Statusへ保存する。
- 保存、採用、保留、Exportの意味を分けて表示する。

## 訓練画像で優先する改善

- 初期3訓練の各5状態を分けた標準15枠の作成から画像採用まで、別タブへ移動せず完了できるようにする。
- `trainingId`、`visualState`、主人公・ヒロインの表示有無を一覧に表示する。
- 訓練画像を選択した時点で対応するPromptと生成対象を自動選択する。
- 採用後は次の未完了枠へ移動できるようにする。

## 実装順序

1. 現行タブへ目的、次の操作、現在状態を表示し、生成ボタンの名称と順序を統一する。
2. 共通の画像制作部品を作り、訓練画像とヒロイン画像から統合する。
3. スチル、プレイヤー、敵を共通部品へ移す。
4. 不足素材とExport警告の確認画面を統合し、修正対象へ移動できるようにする。

既存データ形式とUnity向けExport形式は維持し、画面と操作導線を段階的に変更する。
