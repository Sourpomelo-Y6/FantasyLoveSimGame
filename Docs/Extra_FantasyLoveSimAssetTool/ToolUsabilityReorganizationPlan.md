# AssetTool 操作整理案

AssetToolの操作画面は、機能別タブではなく利用者が完了させたい作業を基準に整理する。
詳細な提案の正本は、AssetToolリポジトリの `Docs/ToolUsabilityReorganizationPlan.md` とする。

## 推奨構成

主タブを次の4つへ整理する。

1. `キャラクター`: ヒロイン、プレイヤー、敵の選択と基本情報
2. `画像制作`: 立ち絵、戦闘、訓練、イベント、行動、エンディングの登録と生成
3. `ゲームデータ`: 会話、イベント、行動反応、表情・衣装・レイヤー定義
4. `確認・Export`: 不足、保留、警告、Export対象と結果

ComfyUI接続設定やworkflow template編集は、日常操作から分離して `設定・詳細` に置く。

## 画像制作の共通手順

```text
対象選択 → 必要画像選択 → Prompt確認 → ComfyUI生成または外部画像指定 → Preview → 採用
```

- 左側に必要画像と登録済み画像をまとめて表示する。
- 中央にPromptと生成操作を表示する。
- 右側に現在画像、生成候補、検査結果、採用操作を表示する。
- 外部登録とComfyUI生成は同じAsset、Prompt JSON、Statusへ保存する。
- 保存、採用、保留、Exportの意味を分けて表示する。

## 訓練画像で優先する改善

- 初期3訓練の各5状態を分けた標準15枠の作成から画像採用まで、別タブへ移動せず完了できるようにする。
- `trainingId`、`visualState`、主人公・ヒロインの表示有無を一覧に表示する。
- 訓練画像を選択した時点で対応するPromptと生成対象を自動選択する。
- 採用後は次の未完了枠へ移動できるようにする。

## キャラクター別制作状況チェック表

機能追加が一通り進んだ段階で、`確認・Export` 内にキャラクター別の制作状況を横断確認するタブページを追加する。

- 行をキャラクター、列を基本情報、各種画像、訓練セリフ、会話、イベント、戦闘スキル、戦闘結果メッセージ、スキルツリー、Export準備などの制作項目とする。
- 必須条件を満たせば `○`、未作成または必須不足なら `×`、部分入力や警告があれば必要に応じて `△`、任意または対象外なら `―` を表示する。
- 記号へマウスポインターを合わせると、判定理由、登録数と必要数、不足ID・不足枠、警告などの詳細をToolTipで表示する。
- セルをクリックすると対象キャラクターを選択し、該当する編集タブ、サブカテゴリ、データ行または入力欄へ移動する。
- 未完成のみの表示と、キャラクター・項目カテゴリの絞り込みに対応する。
- 判定結果は別途保存せず、既存のプロフィール、画像、文章、スキル、Export検証結果から共通サービスで都度集計する。
- 完了条件と移動先は項目定義として管理し、新機能の追加やタブ再編後の移動先変更に対応しやすくする。

セルからの移動は `characterId`、移動先タブ、サブカテゴリ、対象IDを持つ共通ナビゲーション要求として扱い、各タブ同士が直接依存しない構成を推奨する。

AssetToolの `制作状況` タブは現在選択中のキャラクターの基本情報、戦闘メッセージ、訓練画像、会話、表情、衣装、戦闘スキル、スキルツリーを `○ / △ / ×` で判定する。戦闘スキルのID・効果・対象・数値と、ツリーのルート・NodeId・前提・付与スキル・解放訓練参照を個別行で確認できる。カテゴリと詳細行のクリックで対応する編集タブへ移動し、未完成のみの表示と再集計にも対応する。

## 実装順序

1. 現行タブへ目的、次の操作、現在状態を表示し、生成ボタンの名称と順序を統一する。
2. 共通の画像制作部品を作り、訓練画像とヒロイン画像から統合する。
3. スチル、プレイヤー、敵を共通部品へ移す。
4. 不足素材とExport警告の確認画面を統合し、修正対象へ移動できるようにする。
5. キャラクター別制作状況チェック表を追加し、各セルから不足項目へ移動できるようにする。

既存データ形式とUnity向けExport形式は維持し、画面と操作導線を段階的に変更する。
